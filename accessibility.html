1. Compliance Standards

Our portal conforms to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. These guidelines explain how to make web content more accessible to people with disabilities.

Standards We Follow

  • WCAG 2.1 Level AA: International accessibility standard
  • Section 508: U.S. federal accessibility requirements
  • EN 301 549: European accessibility standard
  • Dutch accessibility laws: Local compliance requirements

Regular Auditing

We conduct accessibility audits:

  • Automated testing using industry-standard tools
  • Manual testing by accessibility experts
  • User testing with assistive technology users
  • Quarterly comprehensive accessibility reviews

3. Supported Assistive Technologies

Our portal has been tested and optimized for use with:

Screen Readers

  • NVDA: Windows screen reader (fully supported)
  • JAWS: Professional Windows screen reader (fully supported)
  • VoiceOver: macOS and iOS screen reader (fully supported)
  • TalkBack: Android screen reader (fully supported)
  • Dragon NaturallySpeaking: Voice recognition software

Input Methods

  • Keyboard only navigation: All functionality accessible
  • Voice control: Compatible with voice navigation
  • Switch navigation: Alternative input device support
  • Eye tracking: Compatible with eye-tracking software

Browser Compatibility

  • Chrome (latest version and previous major version)
  • Firefox (latest version and previous major version)
  • Safari (latest version and previous major version)
  • Edge (latest version and previous major version)

5. Content Accessibility

Text Content

  • Reading Level: Clear, concise language appropriate for diverse audiences
  • Structure: Logical heading hierarchy for easy navigation
  • Links: Descriptive link text explaining destinations
  • Instructions: Clear, step-by-step guidance for complex tasks

Images & Graphics

  • Alt Text: Descriptive alternative text for all meaningful images
  • Complex Images: Long descriptions for charts and infographics
  • Decorative Images: Properly marked as decorative for screen readers
  • Text in Images: Avoided where possible, alternatives provided

Forms & Interactive Elements

  • Labels: Clear, associated labels for all form fields
  • Instructions: Helpful guidance for form completion
  • Error Messages: Clear, specific error descriptions
  • Validation: Real-time feedback with accessibility support
